线程池如何保证有序?

发表于 2020-05-03 分类于 java 阅读次数：

背景

在生产环境中，我们会用kafka来解耦是常用的技术手段。为了保证消息的顺序处理，会把相同属性(同一个人、同一个素材等) 的消息发往kafka同一个partition中。例如，在广告系统中，会把某一个ad的转化数据打到同一个partition。

这样做的初衷是希望对于同一个Ad的消息，不要出现消费的时候先后顺序错乱。

但是这里有一个速度不匹配的问题，Consumer处理的速度往往是远远小于Consumer拉取消息的速度的，于是很多人就会想到用多线程加速，但是这会带来一个问题：同一个Ad的消息先后顺序错乱。由此，引出我们今天的问题：线程池如何保证有序。

问题思考

上面我们的思路是通过线程池来加速消息的处理，有一种方法比较简单粗暴且有效，加Partition，增加Consumer，每个Consumer使用单线程来处理。但是如果消息太大，部署成本就会上升，所以回到了线程池如何保证有序上来。我们希望实现这样一个线程池：
